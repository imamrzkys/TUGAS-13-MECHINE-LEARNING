Tentang Proyek

Memahami teknologi Machine Learning di balik deteksi penipuan transaksi digital

Apa Itu Fraud Detection?

Fraud detection atau deteksi penipuan adalah proses identifikasi transaksi yang mencurigakan atau tidak sah dalam sistem keuangan digital. Dengan meningkatnya transaksi digital, kebutuhan akan sistem deteksi fraud yang akurat dan real-time menjadi sangat penting.

Sistem ini menggunakan teknologi Machine Learning untuk menganalisis pola transaksi dan mendeteksi aktivitas yang tidak normal secara otomatis, membantu melindungi pengguna dari kerugian finansial.

Pentingnya Deteksi Fraud

01

Melindungi Pengguna

Mencegah kerugian finansial akibat transaksi penipuan dan melindungi data pengguna

02

Deteksi Real-Time

Mengidentifikasi transaksi mencurigakan secara instan sebelum kerugian terjadi

03

Meningkatkan Kepercayaan

Membangun kepercayaan pengguna terhadap platform transaksi digital

04

Efisiensi Operasional

Mengurangi beban manual review dengan automasi deteksi yang akurat

Cara Kerja Support Vector Machine (SVM)

Support Vector Machine (SVM) adalah algoritma machine learning yang digunakan untuk klasifikasi dengan mencari hyperplane optimal yang memisahkan dua kelas (Normal vs Fraud) dengan margin maksimal.

1

Preprocessing Data

Data transaksi di-preprocess dengan StandardScaler untuk fitur numerik dan OneHotEncoder untuk fitur kategorikal

2

Feature Engineering

7 fitur transaksi (step, type, amount, dan balance) diubah menjadi representasi numerik yang siap untuk model

3

Training Model

Model LinearSVC dilatih dengan data yang sudah di-balance untuk mempelajari pola transaksi normal dan fraud

4

Hyperparameter Tuning

Parameter C di-tune menggunakan GridSearchCV untuk mendapatkan performa optimal

5

Prediksi

Model memprediksi apakah transaksi baru termasuk kategori Normal atau Fraud berdasarkan pola yang dipelajari

Dataset & Performa Model

Sumber Dataset: Penipuan Transaksi Digital
Total Data: ~6.3 juta transaksi
Komposisi: 99.87% Normal, 0.13% Fraud (imbalanced)
Data Training: Balanced 1:1 (16,426 transaksi)
Akurasi Model: ~95%+
Algoritma: Linear Support Vector Classifier (LinearSVC)
Preprocessing: StandardScaler + OneHotEncoder

Kelebihan & Keterbatasan

Kelebihan

  • Akurasi tinggi (~95%) dalam klasifikasi
  • Prediksi sangat cepat (real-time)
  • Efektif untuk dataset besar
  • Robust terhadap overfitting
  • Dapat menangani data non-linear dengan kernel
  • Memberikan decision score untuk interpretasi

Keterbatasan

  • Memerlukan data yang sudah di-balance untuk performa optimal
  • Sensitif terhadap feature scaling
  • Komputasi lebih berat untuk dataset sangat besar
  • Hyperparameter tuning diperlukan untuk hasil terbaik
  • Interpretasi model kurang intuitif dibanding tree-based
  • Memerlukan update berkala untuk adaptasi pola fraud baru
